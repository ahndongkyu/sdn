# SDN 축구 클럽 앱 — 기획서

> Saturday Night (SDN) 조기축구 동호회 전용 관리 앱
> 작성일: 2026-06-30 · 기존 프로젝트(Clear FC) 계승·개선판

---

## 1. 개요 / 컨셉

- **팀명**: SDN (Saturday Night) — 토요일 밤 경기 컨셉
- **대상**: 30대 주축 동호회원 (어르신층 없음 → 세련된 모던 UI 지향)
- **형태**: 웹앱이지만 **앱처럼 사용** (PWA, 하단 고정 5탭)
- **목표**: 기존 Clear FC의 기능(포메이션·매치기록·통계)을 계승하되, 디자인·UX·운영 흐름을 대폭 개선
- **차별점**: 고퀄리티 포메이션 보드, 강화된 팀/개인 통계, 매 경기 유튜브 영상 연동, 참석(RSVP) 중심 운영 흐름

## 3. 회원 / 권한 모델

### 3.1 하이브리드 모델
- 폐쇄형 동호회 → 아무나 가입 불가
- 운영진이 회원 명단(roster)을 등록·관리
- 각 회원은 본인 카카오 계정으로 로그인 → 개인 통계·기록 식별

### 3.2 권한 (role)
- `member` — 조회 + 본인 정보 수정 + 참석 체크 + MVP 투표
- `manager` (운영진) — 매치/회원/결과/참석 관리, 영상 등록
- `admin` — 전체 + 팀 설정

### 3.3 카카오 계정 연결 플로우 (승인 매핑)
```
[운영진] 회원 프로필 등록 (이름·포지션·유니폼별 번호)
            ↓
[회원] 카카오 로그인 → "운영진 승인 대기중" 화면
            ↓
[운영진] 가입 대기 목록 → 해당 회원과 매칭(탭) → 연결 완료
            ↓
[회원] 본인 프로필·기록 사용 가능
```
- 이름 자동매칭은 **사용 안 함** (동명이인·닉네임 불일치 위험)
- 외부인이 로그인해도 승인 전엔 접근 불가 → 폐쇄성 자연 확보

### 데이터 계층 규칙
- **조회(읽기)** → 서버 컴포넌트에서 Supabase 서버 클라이언트로 직접 fetch
- **쓰기(등록·수정)** → Server Action + `revalidatePath`
- **인터랙티브** → `'use client'` (포메이션 D&D, 참석 토글, MVP 투표)
- 세션: `@supabase/ssr` 쿠키 기반 (server / client / middleware)

## 7. 화면 명세

### 7.1 로그인
- SDN 로고(자리) + 카카오 로그인 버튼 1개
- "회원만 이용 · 가입은 운영진 문의" 안내

### 7.2 홈
- **다음 경기 카드(남색)**: D-day, 우리팀 vs 상대, 일시·장소(세로 정렬), 참석 체크 버튼, 포메이션 보기, 참석 현황
- **경기 날씨 카드(분리)**: 경기 시간대 핀포인트 — 기온·상태·강수확률·바람·체감
- **지난 경기 카드**: 결과(스코어) + 🎬 매치 영상 보기

### 7.3 매치 목록
- 세그먼트: 예정 / 지난
- 예정 = 카드(D-day·내 참석상태), 지난 = 승/무/패 칩 + 스코어 행
- 예정·지난 모두 탭 → 매치 상세로 이동
- 운영진: "경기 등록" 버튼

### 7.4 매치 상세
- 결과 헤더(스코어·승/무/패), 일시·장소
- 액션: 🎬 영상 / 포메이션 보기
- **득점 기록**: 우리 득점만 표시 (시간 X), "득점자 — 도움 [이름]" 텍스트
- 오늘의 MVP (투표 결과)
- 참가 선수 (참석/불참 요약 + 명단)
- 운영진: 우상단 "편집" → 결과 입력 / "MVP 투표" 버튼

### 7.5 포메이션 보드
- **야간 조명 피치**(짙은 잔디 + 중앙 상단 조명 글로우 + 흰 라인)
- 포지션별 색 칩 (GK노랑/DF파랑/MF초록/FW주황), 이름 메인 + 등번호 보조
- 프리셋(4-3-3, 4-4-2, 4-2-3-1, 3-5-2) 원터치 + 자유 드래그
- 벤치(선택 가능 선수) / 저장 / 이미지 공유(카톡)
- **선수 풀**: 매치 연결 포메이션 = 참석자만 / 템플릿 포메이션 = 전체 회원

### 7.6 기록
- **팀 시즌 카드(상단)**: 승·무·패·승률, 승무패 비율 바, 총득점·총실점·득실차·경기당 득점
- **개인 랭킹**: 득점왕/도움왕/공격P/출석왕/MVP, 포디움 TOP3 + 리스트(내 순위 하이라이트)
- 시즌 셀렉터, 자체전은 승패 집계 제외
- (개인) 내 시즌 기록 + 최근 5경기 폼

### 7.7 회원정보 (프로필)
- 아바타·이름·닉네임·역할 배지·포지션·주발
- **유니폼별 등번호** (빨검 9 / 흰파 14)
- 시즌 기록(출전·득점·도움·공격P·출석률·MVP) + 최근 5경기 폼
- 본인/운영진만 편집

### 7.8 더보기
- 상단 내 프로필 미니
- 일반: 내 프로필·회원목록·공지사항·회비(**준비중** 잠금)
- 운영진 전용(빨강 테두리 구분): 매치 관리·회원 관리·팀 설정
- 설정·로그아웃

### 7.9 회원 목록
- 포지션별 그룹(FW/MF/DF), 각 회원 유니폼별 번호 표시, 운영진 배지, 검색
- 운영진: "회원 등록"

### 7.10 매치 등록 (운영진)
- 경기 유형(매치/자체전), 상대팀, 날짜/시간, 장소(날씨 연동), **유니폼 선택**, 유튜브 URL(경기 후)

### 7.11 회원 등록 (운영진)
- 이름, 주/부 포지션, 주발, **유니폼별 등번호**(추가 가능), 권한
- 안내: "등록 후 회원이 카카오 로그인하면 운영진이 승인·연결"

### 7.12 결과 입력 (운영진)
- **최종 스코어**: 양쪽 동일한 −/+ 스테퍼로 직접 입력 (대칭)
- **우리 득점 상세(통계용·선택)**: 득점자/도움 칩 선택 → 추가 ("N/스코어 입력됨" 비교 표시)
- 스코어와 득점 상세는 강제 일치 X (동호회 현실 반영)
- 득점자/도움 후보 = **참석자만**

### 7.13 MVP 투표 (회원)
- 경기 정보 + "1인 1표 · 마감 후 공개 · 참여율"
- 오늘 뛴 선수(참석자) 중 1명 선택 → 투표
- 운영진이 마감 → 매치 상세 "오늘의 MVP"에 반영

### 7.14 참석 관리 (운영진)
- 참석/불참/미정 요약
- 전 회원 행에 참/불/미 토글 → **운영진 대리 체크**
- 출처 라벨: 본인 / 운영진 대리 / 미체크(경고)
- **게스트 추가** (명단 외 용병)

### 7.15 가입 승인 대기 (운영진)
- 카카오 로그인했으나 미연결 회원 목록 → 등록된 프로필과 매칭

## 9. Supabase 데이터 모델

```
profiles
  id (uuid, auth 연동), kakao_id, name, nickname, role(member/manager/admin),
  position1, position2, foot(L/R/both), photo_url, status, linked(bool)

uniforms
  id, team_id, name(빨검/흰파/연핑크/진남색), color

member_numbers           # 유니폼별 등번호
  member_id, uniform_id, number

matches
  id, date, time, place, place_coord(위경도, 날씨용),
  opponent, type(매치/자체전), uniform_id(이 경기 유니폼),
  score_for, score_against, youtube_url, mvp_member_id, status

attendances              # 참석 RSVP (핵심 허브)
  match_id, member_id, status(참석/불참/미정), source(self/manager), is_guest

players                  # 출전 기록 (쿼터 단위)
  match_id, member_id, quarter, status(출전/휴식/교체)

goals                    # 우리 득점 상세 (통계용)
  match_id, member_id, assist_member_id(nullable), is_own_goal(상대 자책)

formations
  id, match_id(nullable=템플릿), name, layout(jsonb: 선수별 x,y% + 벤치)

mvp_votes
  match_id, voter_id, target_member_id

notices
  id, title, content, author_id, created_at

# 2차: dues(회비), guests 상세 등
```

### 참석(attendances)이 핵심 허브
참석자 명단 하나로 아래가 전부 결정됨:
- 포메이션 보드 선수 풀
- 결과 입력의 득점자/도움 후보
- MVP 투표 후보

### 엣지 케이스
- **게스트/용병**: attendances에 `is_guest`로 임시 추가 → 득점자 후보 포함
- **자책골(상대 OG)**: 스코어만 +1, `goals.is_own_goal` 또는 득점자 미입력
- 참석 안 한 회원은 후보에서 자동 제외 (실수 방지)

## 10. 권한 (RLS)

- Supabase **Row Level Security**로 DB 레벨 강제
  - 일반 회원: 읽기 + 본인 attendances/mvp_votes/profile 쓰기
  - 운영진: matches/members/goals/결과/참석 쓰기
- 프론트: 운영진 메뉴·버튼 조건부 노출 (이중 방어)

## 11. 외부 연동

- **카카오 로그인**: Supabase Auth OAuth provider
- **날씨**: Open-Meteo REST — 경기장 좌표 + 날짜/시간으로 조회
  - 예보는 7~10일 전부터 정확 → 그 이전은 "경기일 가까워지면 표시" 플레이스홀더
  - 장소에 좌표 매핑 저장
- **유튜브**: 매치에 `youtube_url` 저장, 매치 상세/홈에서 버튼 → 영상 연결 (운영진만 등록)

## 13. 개발 순서 (로드맵)

1. **기반 세팅** — Next.js + Tailwind + shadcn + Supabase 연결, PWA manifest, 색 토큰, 하단 5탭 셸
2. **인증** — 카카오 로그인, profiles, 역할, 승인 매핑, RLS
3. **회원** — 등록 폼, 목록, 프로필, 유니폼/등번호
4. **매치** — 등록, 목록, 상세, 유니폼 선택
5. **참석 RSVP** — 참석 체크, 운영진 관리, 게스트
6. **포메이션 보드** — dnd-kit + SVG 피치, 프리셋, 저장, 이미지 공유
7. **결과·기록** — 결과 입력, 득점 상세, 팀/개인 통계 집계(SQL View/RPC)
8. **MVP 투표** — 투표·마감·반영
9. **홈 대시보드** — 다음 경기, 날씨(Open-Meteo), 지난 경기+영상
10. **마감** — PWA 점검, 배포(Vercel+Supabase)
11. **2차** — 회비·공지·푸시 알림·자동 팀 나누기
